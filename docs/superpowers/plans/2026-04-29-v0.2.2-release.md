# v0.2.2 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare and verify the `v0.2.2` release with private real-subscription validation first, then GitHub-hosted Linux, macOS, and Windows prebuilt packages.

**Architecture:** Keep real subscription validation local and transient so private URLs never enter source, docs, CI, release notes, package metadata, or artifacts. Use existing CMake/CPack installation rules for packaged assets, add platform-specific CPack generators, and add a GitHub Actions release workflow that builds/tests/packages each OS from tag `v0.2.2`.

**Tech Stack:** C++17, CMake, CPack, CTest, GitHub Actions, shell/PowerShell release jobs

---

## File Responsibility Map

- `CMakeLists.txt`: release version, platform package generator selection, deterministic package names.
- `.github/workflows/release.yml`: GitHub Actions matrix for Linux, macOS, Windows build/test/package and tag-based release asset upload.
- `docs/superpowers/plans/2026-04-29-v0.2.2-release.md`: this implementation and verification plan. It must not contain private subscription URLs.
- No source file should persist the private test subscription URLs.
- No test fixture, CI secret, workflow variable, release note, or package file should persist the private test subscription URLs.

### Task 4: Add GitHub Actions release workflow

**Files:**
- Create: `.github/workflows/release.yml`
- Test: YAML review plus local package verification

- [ ] **Step 1: Create the workflow**

Create `.github/workflows/release.yml` with this content:

```yaml
name: Release

on:
  workflow_dispatch:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  build:
    name: Build ${{ matrix.name }}
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        include:
          - name: linux-x86_64
            os: ubuntu-latest
          - name: macos-arm64
            os: macos-latest
          - name: windows-x86_64
            os: windows-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Configure
        run: cmake -S . -B build -DCMAKE_BUILD_TYPE=Release

      - name: Build
        run: cmake --build build --config Release -j

      - name: Test
        run: ctest --test-dir build --build-config Release --output-on-failure

      - name: Package
        run: cmake --build build --config Release --target package

      - name: Upload package artifact
        uses: actions/upload-artifact@v4
        with:
          name: subcli-${{ matrix.name }}
          path: |
            build/subcli-*.tar.gz
            build/subcli-*.zip
          if-no-files-found: error

  release:
    name: Publish GitHub Release
    needs: build
    if: startsWith(github.ref, 'refs/tags/')
    runs-on: ubuntu-latest
    steps:
      - name: Download package artifacts
        uses: actions/download-artifact@v4
        with:
          path: dist
          merge-multiple: true

      - name: Publish release
        uses: softprops/action-gh-release@v2
        with:
          files: dist/*
          generate_release_notes: true
```

- [ ] **Step 2: Verify workflow contains no private subscription data**

Run:

```bash

```

Expected: no private subscription URL or token is present. The generic word `subscription` should not appear unless unrelated release prose is later added.
